PLANNED FEATURES
----------------

[FEATURE] (todo) Share checklist
    Let the user share a checklist with others (e.g. via the native share
    sheet, a link, or export). Should include the todo text, dates, and
    checked state.

[FEATURE] (done) Checklist groups (todo inside todo list)
    A todo can contain a checklist of sub-tasks (e.g. "Shopping" -> milk,
    eggs). Edit a todo and use "Checklist items" to add/rename/remove items.
    The list shows a progress count (e.g. ✓ 2/5) and an expand arrow to
    reveal items; each item has its own checkbox. Checking the parent checks
    all items; the parent auto-checks when all items are done (and shows a
    dash when partially done).
    Implementation: SubTask type on Todo; see src/components/TodoEditModal.tsx
    (editor), src/components/TodoItem.tsx (display), App.tsx (toggle logic).
    DECISION: items are added MANUALLY for now (voice still creates top-level
    todos). Voice-to-subitems could be a follow-up.

[FEATURE] (todo) Assing a todo to some person.
    User should be able to assing a Todo to some person. 
    For example: Get cups -> Marko, Buy bread -> Pave

[FEATURE] (done) Restore deleted todo (undo)
    When the user deletes a todo, give them the option to restore it.
    Implemented as an "Undo" snackbar shown for ~4.5s after deletion;
    tapping UNDO restores the todo to its original position.
    See src/components/UndoSnackbar.tsx and App.tsx.


[FEATURE] (done) Group / filter todos by date
    List is grouped into sections (Overdue, Today, Tomorrow, then each
    future date) with a filter bar (All / Overdue / Today / Upcoming,
    each showing a count). See src/grouping.ts and App.tsx.
    Note: distinct from named "Shopping"-style groups above.


[FEATURE] (done) Give alert on the day of the todo
    Per-todo opt-in reminder: in the edit dialog, a "Remind me on this day"
    switch + a time picker. Schedules a local notification on the todo's
    date at the chosen time (expo-notifications). A 🔔 HH:MM chip shows on
    the todo. Reminders are cancelled/rescheduled when the todo is completed,
    edited, or deleted. See src/notifications.ts, TodoEditModal.tsx, App.tsx.
    NOTE: requires a native rebuild (expo-notifications is a native module).
