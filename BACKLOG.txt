VOICE TODO — FEATURE & BUG BACKLOG

How to use this file:
- Add one entry per feature or bug.
- Prefix each with a tag: [FEATURE] or [BUG].
- Optionally add a status: (todo) / (in progress) / (done).
- Keep newest ideas at the top.


PLANNED FEATURES
----------------

[FEATURE] (todo) Share checklist
    Let the user share a checklist with others (e.g. via the native share
    sheet, a link, or export). Should include the todo text, dates, and
    checked state.

[FEATURE] (todo) Checklist groups (todo inside todo list)
    Support grouping todos into named checklists (e.g. "Shopping", "Work",
    "Trip"). Each group has its own list of items. User can create, rename,
    and delete groups, and add todos to a specific group.
    After user creates a todo, he can edit it and create a list inside that todo. 
    For example: Todo for Shoping then inside that todo he can add a list of items to buy like milk, egs, etc...
    SHould the user add it manually or can it be done over voice?

[FEATURE] (todo) Assing a todo to some person.
    User should be able to assing a Todo to some person. 
    For example: Get cups -> Marko, Buy bread -> Pave

[FEATURE] (done) Restore deleted todo (undo)
    When the user deletes a todo, give them the option to restore it.
    Implemented as an "Undo" snackbar shown for ~4.5s after deletion;
    tapping UNDO restores the todo to its original position.
    See src/components/UndoSnackbar.tsx and App.tsx.


[FEATURE] (done) Group / filter todos by date
    List is grouped into sections (Overdue, Today, Tomorrow, then each
    future date) with a filter bar (All / Overdue / Today / Upcoming,
    each showing a count). See src/grouping.ts and App.tsx.
    Note: distinct from named "Shopping"-style groups above.
